### f-094 missing_relationship -- Fact - Sales[Order Date] -> Date[Date]
- **Category:** relationships
- **Object:** Fact - Sales[Order Date] -> Date[Date] (relationship)
- **Auto-fixable:** Yes

> No relationship exists between 'Fact - Sales'[Order Date] and 'Date'[Date]. Only 'Delivery Date' is connected to the Date table. Order Date is likely the primary date for most analyses but has no relationship to the Date dimension.

**Recommendation:** Create a relationship from 'Fact - Sales'[Order Date] to 'Date'[Date]. Per Power BI Standards (Data Modeling > Dates): 'If possible use one date table if there are multiple date fields and use the function USERELATIONSHIP instead of creating additional date tables.' Make Order Date the active relationship and Delivery Date inactive, then use USERELATIONSHIP in measures that need Delivery Date context.

### f-096 measure_logic -- ⍟ Big & Bulky Measures[Total Cost]
- **Category:** measures_calculations
- **Object:** ⍟ Big & Bulky Measures[Total Cost] (measure)
- **Auto-fixable:** Yes

> Measure 'Total Cost' uses SUM('Fact - Sales'[Unit Cost]) which sums unit costs without multiplying by quantity. This likely produces incorrect results. Compare with 'Total Revenue' which correctly uses SUMX with quantity multiplication.

**Recommendation:** Change to: SUMX('Fact - Sales', 'Fact - Sales'[Unit Cost] * 'Fact - Sales'[Quantity]) to calculate the actual total cost accounting for quantity purchased.

### f-109 date_table_filtered -- Date
